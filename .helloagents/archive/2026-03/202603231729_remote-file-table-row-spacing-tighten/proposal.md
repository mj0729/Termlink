## 1. 需求

### 背景
远程文件列表的表头和底部状态栏已经收紧，但正文每一行的垂直占位仍偏松。用户明确要求这次只压列表每一行的高度，让整个文件列表更紧凑，不额外调整图标尺寸、文字字号或名称列内部图标间距。

### 目标
- 收紧远程文件表格 compact 模式下的正文行高。
- 保持文件名、图标、列数据的可读性与现有交互不变。
- 限制改动在文件列表正文，不影响表头和底部状态栏。

### 约束条件
```yaml
时间约束: 本轮直接完成
性能约束: 不增加运行时逻辑或依赖
兼容性约束: 保持 vxe-table 的选择、拖拽、双击打开与排序行为
业务约束: 仅压缩每行高度，不改图标尺寸和文字字号
```

### 验收标准
- [ ] compact 模式下文件列表正文每行视觉高度明显收紧
- [ ] 图标和文字仍保持当前大小与可读性
- [ ] `pnpm run build` 通过

---

## 2. 方案

### 技术方案
下调 `RemoteFileTable.vue` 中 compact 模式的 `rowHeight`，并同步让名称单元跟随新的 `--remote-table-row-height` 自动收紧。保留现有图标尺寸、字号和列布局，仅通过行高变量收口正文垂直间距。

### 影响范围
```yaml
涉及模块:
  - ui-components: 远程文件工作台正文列表行高收紧
预计变更文件: 1
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 行高压缩后长文件名显得拥挤 | 低 | 只下调 compact 行高一档，保留图标与字号不变 |
| 影响选中/拖拽热区体验 | 低 | 继续通过 row-config 与名称单元最小高度维持可点击区域 |

## 5. 技术决策

> 本方案涉及的技术决策，归档后成为决策的唯一完整记录

### remote-file-table-row-spacing-tighten#D001: 仅下调 compact 正文行高，不联动图标与字号
**日期**: 2026-03-23
**状态**: ✅采纳
**背景**: 用户明确要求只压每一行的高度，不希望这轮同时改变图标大小或名称列内部间距。
**选项分析**:
| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 只下调 compact `rowHeight` | 改动最小，命中用户诉求 | 收紧幅度有限 |
| B: 同时缩小图标、字号和名称间距 | 紧凑效果更强 | 超出当前确认范围 |
**决策**: 选择方案 A
**理由**: 本轮目标是“压行高”，不是重做内容密度；最稳妥的是先收正文行高变量。
**影响**: 仅影响 `RemoteFileTable.vue` 正文列表密度
