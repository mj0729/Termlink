## 1. 需求

### 背景
当前工作台顶部、底部、左右侧栏同时承载了连接状态、标签计数、主题信息和空状态提示，造成信息重复、控件偏大、边框层级过多，小屏下主内容区被明显挤压。

### 目标
- 去掉外层和左右分栏的硬边框感，让工作台直接融入背景
- 收敛重复信息，减少顶部/底部/空状态之间的重复表达
- 压缩顶部操作、标签栏、连接卡片和右侧监控面板的控件尺寸
- 提升窄屏宽度下的空间利用率，优先保证终端与文件内容区可用面积

### 约束条件
```yaml
技术约束:
  - 保持现有 Vue 3 + antdv-next + Tauri 技术栈
  - 不改动 SSH / SFTP / 下载 / 终端底层逻辑
体验约束:
  - 保留现有工作台三栏结构与折叠行为
  - 浅色与深色主题都要成立
工具约束:
  - 使用 antdv-next 现有组件体系完成布局压缩
  - better-icons CLI 本机不可用时允许回退到现有 @antdv-next/icons
```

### 验收标准
- [x] 顶部、底部、空状态中的重复状态信息明显减少
- [x] 外层容器与左右分栏不再依赖明显边框分割
- [x] 顶栏、标签页、侧栏卡片和右栏控件高度整体压缩
- [x] `pnpm run build` 通过

## 2. 方案

### 技术方案
采用“背景一体化 + 信息收敛 + 控件降密度”的统一重构路径：
1. 先重写全局主题变量与工作台外壳，让背景、面板和内容区使用同一套半透明层级。
2. 顶部工具带改为单行上下文结构，只保留导航、当前连接和快捷动作。
3. 底部状态条改为轻量提示，不再重复展示主题与标签统计。
4. 左右侧栏卡片和标签栏改为更薄的胶囊/磨砂层，减弱边框和阴影。
5. 空状态重写为双列介绍卡片，直接体现新的紧凑布局方向。

### 影响范围
```yaml
涉及模块:
  - app-shell: 整体工作台容器、空状态、全局主题变量
  - ui-navigation: 顶部栏、标签栏、底部状态条
  - ui-panels: 左侧连接管理、右侧监控/下载栏
预计变更文件: 7
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 半透明背景过多导致深浅主题对比不足 | 中 | 同步调整 light/dark 变量，保留关键内容区域的文字和操作对比 |
| 样式压缩后局部可点击区域过小 | 中 | 保持按钮最小高度 34px，交互区仍高于文本行高 |
| 只改视觉层可能遗漏局部重复信息 | 低 | 同步收敛 TopMenu / StatusBar / 空状态文案与统计信息 |

## 3. 成果设计

### 设计方向
- **美学基调**: 更像“融入背景的桌面工作台”，而不是被多层卡片切开的控制台
- **记忆点**: 顶部与侧栏都变成轻薄玻璃层，终端内容区获得更完整的视觉焦点
- **布局立场**: 小屏优先，所有辅助信息让位给主要内容

### 视觉要素
- **配色**: 保留蓝色主强调，但把大面积纯白卡片换成更轻的半透明层
- **布局**: 移除外围壳体边框，用间距和模糊背景区分层次
- **组件风格**: 大胶囊压缩为短胶囊，卡片边框改为柔和内描边或直接取消
- **动效**: 沿用现有 hover / active 微动效，不新增复杂动画

## 4. 关键决策

### compact-workspace-redesign#D001: 以“信息收敛”优先于“单纯缩小控件”
**日期**: 2026-03-18
**状态**: ✅采纳
**背景**: 用户反馈的不只是尺寸大，还包括页面信息重复和工作台被边框切碎。
**决策**: 先删重复状态，再统一缩小控件和边界层次。
**影响**: 直接影响 `TopMenu.vue`、`StatusBar.vue`、`App.vue` 的信息结构。

### compact-workspace-redesign#D002: better-icons 不可用时回退到现有图标体系
**日期**: 2026-03-18
**状态**: ✅采纳
**背景**: 用户希望使用 `better-icons`，但本机命令不可执行。
**决策**: 不阻塞主线改造，回退到 `@antdv-next/icons` 和文本型标签。
**影响**: 图标风格保持一致，不额外引入 SVG 资产与构建风险。
