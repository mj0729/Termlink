## 1. 需求

### 背景
远程文件工作台的文件表头、正文行距和状态栏已经逐步收紧，但左侧目录树在 aggressive 模式下仍显得偏松。用户明确要求本轮只压目录节点的行高和上下留白，不调整字体大小。

### 目标
- 收紧左侧目录树节点的纵向高度。
- 保持当前字体大小、图标大小和层级结构不变。
- 限制改动在 SSH 工作区左树的 aggressive 紧凑模式。

### 约束条件
```yaml
时间约束: 本轮直接完成
性能约束: 不增加运行时逻辑或依赖
兼容性约束: 保持树节点选择、展开、拖拽和重命名行为不变
业务约束: 不改字体大小，只压节点行高和上下留白
```

### 验收标准
- [ ] aggressive 模式下左侧目录节点视觉高度进一步收紧
- [ ] 字体大小、图标大小和缩进保持当前水平
- [ ] `pnpm run build` 通过

---

## 2. 方案

### 技术方案
通过 `RemoteFileWorkbench.vue` 中的 aggressive 样式变量下调目录树节点的最小高度，并同步收紧树内联编辑输入框高度，让目录树节点垂直占位变矮但不改变字体、图标和缩进尺寸。

### 影响范围
```yaml
涉及模块:
  - ui-components: SSH 工作区左侧目录树节点密度收紧
预计变更文件: 1
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 节点过矮导致点击热区下降 | 低 | 只收一档，保留当前字体和图标尺寸 |
| 重命名输入框与节点高度不一致 | 低 | 同步调整输入框高度以匹配节点 |

## 5. 技术决策

> 本方案涉及的技术决策，归档后成为决策的唯一完整记录

### remote-directory-tree-row-spacing-tighten#D001: 仅下调 aggressive 目录树节点高度变量
**日期**: 2026-03-23
**状态**: ✅采纳
**背景**: 用户明确只希望压节点行高和上下留白，不希望同时改变字体大小。
**选项分析**:
| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 只下调 aggressive 节点最小高度和输入框高度 | 改动最小，符合要求 | 收紧幅度有限 |
| B: 同时压缩字体、图标和缩进 | 整体更紧凑 | 超出本轮确认范围 |
**决策**: 选择方案 A
**理由**: 用户要求聚焦行高与上下留白，使用变量下调是最稳妥的路径。
**影响**: 仅影响 `RemoteFileWorkbench.vue` 中目录树 aggressive 变量
