## 1. 需求

### 背景
当前连接中心已经具备搜索、筛选和卡片启动能力，但整体仍偏“基础入口页”：工作区外层氛围较淡，信息层级不够清晰，卡片的资产感和操作感不足，难以承接专业 SSH 工作台的第一印象。

### 目标
- 把连接中心重塑为更高级、简约、专业的连接工作台
- 统一连接中心主内容区与所在工作区外层的视觉语言
- 提升连接卡片的识别度、层级感和可操作性
- 保持现有交互语义与数据结构，不引入额外业务复杂度

### 约束条件
```yaml
技术约束:
  - 保持现有 Vue 3 + antdv-next + Tauri 技术栈
  - 不改动 SSH 连接启动、标签页管理和配置存储逻辑
体验约束:
  - 保留“搜索 + 分组筛选 + 连接卡片 + 新建动作”的工作流
  - 浅色与深色主题都要成立
组件约束:
  - 延续 antdv-next 的 Input / Segmented / Card / Tag 组件骨架
  - 允许通过自定义样式强化视觉质感，但不脱离现有设计体系
```

### 验收标准
- [x] 连接中心头部、筛选区、统计区与卡片区形成清晰的三层信息结构
- [x] 工作区外层与连接中心内部形成统一的高级简约控制台氛围
- [x] 连接卡片能清楚展示名称、主机、认证方式、分组与标签信息
- [x] `pnpm run build` 通过

---

## 2. 方案

### 技术方案
采用“外层场景化 + 内层资产化”的一体化重构路径：
1. 在工作区外层为连接中心增加更克制的渐变背景、柔和高光和边界控制，让入口页具备独立场景感。
2. 重写 `ConnectionHub.vue` 的头部结构，把原本松散的标题、搜索、统计重新组织为工作台首页式布局。
3. 将连接卡片升级为更像“连接资产单元”的组件，强化主机身份、认证方式、分组标签和连接动作提示。
4. 保留原有数据筛选逻辑，仅补充更适合展示层的派生信息与文案。

### 影响范围
```yaml
涉及模块:
  - ui-components: ConnectionHub 入口页结构与样式
  - app-shell: 连接中心所在工作区外层背景与容器层次
预计变更文件: 4
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 新增视觉层后浅色主题显得过亮 | 中 | 使用低饱和蓝灰与透明白叠层，控制大面积纯白 |
| 深色主题下卡片边界和信息对比不足 | 中 | 同步设计 dark 变量下的描边、辉光和文字层级 |
| 结构重排后移动端换行过多 | 低 | 为头部与工具区增加明确断点和纵向堆叠策略 |

## 3. 成果设计

### 设计方向
- **美学基调**: 精致、克制、专业，像桌面端运维工具的连接资产首页
- **记忆点**: 工作区外层有轻场景感，内部卡片像可立即激活的连接资产
- **布局立场**: 不是营销页，而是面向高频使用者的专业工作台入口

### 视觉要素
- **配色**: 以冷静蓝灰和云白为主，保留主蓝色作为动作与状态强调
- **布局**: 头部采用左右双栏，左侧承载定位与概览，右侧承载搜索与快速动作
- **组件风格**: 胶囊筛选 + 柔和玻璃面板 + 具有资产卡片感的连接单元
- **动效**: 使用轻微 hover 抬升、边界发光和背景渐显，不引入喧宾夺主的复杂动画

## 4. 关键决策

### connection-center-workbench-polish#D001: 连接中心升级为“工作台入口”而不是单纯列表页
**日期**: 2026-03-19
**状态**: ✅采纳
**背景**: 用户明确要求高级、简约、美观、大方、专业，单纯微调圆角和配色不足以支撑目标。
**决策**: 重组头部与卡片信息层级，同时联动工作区外层氛围，形成完整首页体验。
**影响**: 直接影响 `src/components/ConnectionHub.vue` 与 `src/App.vue`。

### connection-center-workbench-polish#D002: 继续沿用 antdv-next 组件骨架，通过样式定制提升质感
**日期**: 2026-03-19
**状态**: ✅采纳
**背景**: 项目已完成 antdv-next 统一，新的入口页应保持生态一致，不再引入另一套组件体系。
**决策**: 保留 `a-input`、`a-segmented`、`a-card`、`a-tag` 等骨架，只增强结构与视觉。
**影响**: 避免引入新的交互依赖和兼容成本。
