## 1. 需求

### 背景
上一轮已把 SSH 工作区 aggressive 模式下左侧目录树节点高度从 `19px` 收到 `17px`，但用户继续反馈“还能再紧一点”。本轮要求只再压节点行高一档，不调整缩进、图标大小和字体大小。

### 目标
- 将 aggressive 模式下左侧目录树节点高度再收一档。
- 保持当前缩进、图标大小和字体大小不变。
- 保证选择、展开、拖拽和重命名输入框仍与节点高度匹配。

### 约束条件
```yaml
时间约束: 本轮直接完成
性能约束: 不增加运行时逻辑或依赖
兼容性约束: 保持树节点交互行为和当前视觉语言稳定
业务约束: 仅继续压节点行高，不改缩进、图标和字号
```

### 验收标准
- [ ] aggressive 模式下左侧目录树节点视觉高度继续收紧
- [ ] 缩进、图标大小和字体大小保持当前水平
- [ ] `pnpm run build` 通过

---

## 2. 方案

### 技术方案
继续通过 `RemoteFileWorkbench.vue` 中的 aggressive 变量下调目录树节点最小高度、节点行高和重命名输入框高度，让节点再矮一档，但不改变缩进、图标和字号。

### 影响范围
```yaml
涉及模块:
  - ui-components: SSH 工作区左侧目录树进一步紧凑化
预计变更文件: 1
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 节点继续压缩后点击热区略降 | 低 | 只再收一档，并同步输入框高度 |
| 目录名显得拥挤 | 低 | 保持字体大小和缩进不变，避免同时压多个维度 |

## 5. 技术决策

> 本方案涉及的技术决策，归档后成为决策的唯一完整记录

### remote-directory-tree-row-spacing-tighten-more#D001: 只继续下调 aggressive 节点高度链一档
**日期**: 2026-03-23
**状态**: ✅采纳
**背景**: 用户确认方向正确，但希望左树再更紧一些，同时不希望联动缩进、图标或字体。
**选项分析**:
| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 仅继续下调节点高度链 | 改动最小，延续现有方向 | 收紧幅度有限 |
| B: 同时压缩缩进和图标 | 紧凑效果更强 | 超出本轮确认范围 |
**决策**: 选择方案 A
**理由**: 当前最稳的路径是顺着上一轮继续收节点高度，而不改变树的结构感。
**影响**: 仅影响 `RemoteFileWorkbench.vue` 中 aggressive 目录树变量
