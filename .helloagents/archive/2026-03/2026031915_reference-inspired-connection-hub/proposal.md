## 1. 需求

### 背景
当前连接中心虽然已经比初版更完整，但仍偏重展示型首页，和用户偏好的“扁平、工具感、克制、偏桌面客户端”的参考界面存在明显差距。

### 目标
- 保留现有连接中心功能与数据结构
- 借鉴参考图的视觉语言，把连接中心收敛为更扁平的工具界面
- 降低发光、渐变和大块面板感，增强工具条与主列表气质
- 让连接卡片更接近轻量主机列表项，而不是首页营销卡片

### 约束条件
```yaml
技术约束:
  - 保持现有 Vue 3 + antdv-next + Tauri 技术栈
  - 不修改 SSH 启动、筛选逻辑和标签页交互
布局约束:
  - 保留当前连接中心的搜索、分组筛选、推荐入口和卡片列表能力
  - 不直接复制参考图的左侧导航结构
视觉约束:
  - 只借鉴视觉语言与布局气质，不做像素级复刻
```

### 验收标准
- [x] 连接中心头部更像工具条和工作区，而不是大首页 Hero
- [x] 工作区背景和面板边界更克制，减少发光和套盒感
- [x] 连接卡片高度、层级和信息组织更接近轻量列表卡片
- [x] `pnpm run build` 通过

---

## 2. 方案

### 技术方案
采用“视觉语言迁移，而非结构照搬”的方式：
1. 把当前大 Hero 收成更扁平的页头和工具区，弱化装饰背景。
2. 将搜索、推荐入口和操作按钮压成更接近桌面客户端顶部工具条的结构。
3. 把连接卡片改为更低高度的轻量列表卡，减少中心化大留白。
4. 调整连接中心所在工作区背景和边界，减弱此前的 scene 感。

### 影响范围
```yaml
涉及模块:
  - ui-components: ConnectionHub 结构与样式
  - app-shell: 连接中心所在工作区背景与边界层次
预计变更文件: 4
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 收得过平导致页面没有重点 | 中 | 保留轻量标题区与推荐入口，但控制在工具界面语言内 |
| 扁平化后信息密度过高 | 低 | 通过字号、留白和分组线建立层次，而不是靠大卡片 |
| 与现有浅色主题变量不协调 | 中 | 同步收敛工作区背景和描边变量 |

---

## 3. 成果设计

### 设计方向
- **美学基调**: 扁平、克制、专业，偏桌面客户端工具感
- **记忆点**: 页头更像工作区标题和工具条，连接列表更像主机清单
- **布局立场**: 少装饰、多结构；少首页感、多工具感

### 视觉要素
- **配色**: 继续使用冷静蓝灰，但降低渐变和发光占比
- **布局**: 页头左标题右搜索，下面是轻量控制条，再进入连接列表
- **组件风格**: 保留 antdv-next 骨架，但统一成更薄、更平、更贴近原生客户端的形态
- **动效**: 仅保留轻微 hover，去掉明显抬升和发光存在感
