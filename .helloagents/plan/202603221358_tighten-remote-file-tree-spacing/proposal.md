## 1. 需求

### 背景
当前远程文件管理界面采用偏圆润的容器和树节点样式，左侧目录树的宽度、节点高度、缩进和上下留白都比较松散。实际使用时可见目录数量偏少，视觉上也与用户期望的更利落、工具化文件管理器风格不一致。

### 目标
- 去掉远程文件工作台内容区域的圆角，恢复更硬朗的文件管理器观感。
- 将左侧目录树调整为“中度收紧”密度，减少无效留白并提升同屏可见节点数。
- 保持当前交互逻辑、主题兼容性和可读性，不引入结构性重构。

### 约束条件
```yaml
时间约束: 本次改动在现有组件内完成，不扩展为交互重构
性能约束: 仅做样式层调整，不增加额外渲染逻辑
兼容性约束: 需同时兼容现有浅色/深色主题与 compact/aggressive 密度模式
业务约束: 不修改文件树加载、拖拽、重命名、表格数据展示逻辑
```

### 验收标准
- [ ] 远程文件工作台内容区域不再显示圆角，边缘呈直角风格。
- [ ] 左侧目录树的容器内边距、节点高度、节点缩进和间距均明显收紧，但文字与图标仍清晰可读。
- [ ] 表格区与路径栏在视觉上保持协调，不出现明显错位或过度拥挤。
- [ ] 项目可以通过前端构建验证。

## 2. 方案

### 技术方案
基于现有 `RemoteFileWorkbench.vue`、`RemoteDirectoryTree.vue` 和必要的路径栏/表格样式做局部 CSS 调整，不触碰数据流或事件逻辑。

具体策略：
- 在工作台容器层移除 aggressive 模式下的内容圆角。
- 收紧目录树容器宽度和内边距，减少左右留白。
- 下调树节点包裹层最小高度、节点标题内边距、图标与文字间距，并覆盖 Ant Tree 默认缩进以提升密度。
- 视情况同步微调路径栏与表格行的圆角/内边距，让整体视觉更统一。

### 影响范围
```yaml
涉及模块:
  - RemoteFileWorkbench: 调整工作台内容区域与高密度模式的外观变量
  - RemoteDirectoryTree: 调整目录树容器、节点、缩进与 hover/current 态样式
  - RemotePathBar: 微调工具栏与面包屑紧凑度，避免与树区密度失衡
  - RemoteFileTable: 视需要统一文件名单元格与输入框的圆角/密度
预计变更文件: 4
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 目录树过度压缩导致可读性下降 | 低 | 采用中度收紧方案，优先减少无效留白而不是极限压缩 |
| 不同主题下高亮态观感变差 | 低 | 复用现有颜色变量，仅调整尺寸和圆角 |
| Ant Tree 默认样式覆盖不完整 | 中 | 使用局部 `:deep()` 覆盖关键节点类并通过构建检查样式作用域 |

## 5. 技术决策

> 本方案涉及的技术决策，归档后成为决策的唯一完整记录

### tighten-remote-file-tree-spacing#D001: 采用局部样式压缩而非重写文件树结构
**日期**: 2026-03-22
**状态**: ✅采纳
**背景**: 用户希望快速去掉文件管理圆角并让目录树更紧凑，但当前问题集中在视觉密度与样式参数，而不是结构或交互能力不足。
**选项分析**:
| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 局部样式调整 | 改动小、风险低、能快速命中圆角和留白问题 | 只能在现有结构内优化 |
| B: 重写树组件结构 | 可完全自定义密度与视觉细节 | 成本高，会带来拖拽、重命名和选中态回归风险 |
**决策**: 选择方案A
**理由**: 当前诉求明确且局部，样式参数已经足以解决问题，没有必要引入结构性重构。
**影响**: 主要影响远程文件工作台相关前端组件样式，不影响后端和交互协议

## 6. 成果设计

> 含视觉产出的任务由 DESIGN Phase2 填充。非视觉任务整节标注"N/A"。

### 设计方向
- **美学基调**: 工具化桌面风格。减少圆润修饰，强调直线边界、紧凑排布和更高的信息密度，让文件管理区更像专业桌面文件浏览器。
- **记忆点**: 左侧树区由松散圆润切换为更利落的直角窄边栏，同屏可见层级明显增多。
- **参考**: 用户提供截图，重点问题为外层圆角、树节点垂直空白和容器左右留白过大。

### 视觉要素
- **配色**: 延续现有浅/深色主题变量，不做配色换代，重点通过尺寸和边界风格提升工具感。
- **字体**: 保持项目既有字体栈，避免在本次局部密度调整中引入额外视觉变量。
- **布局**: 保持“路径栏 + 左树右表 + 底栏”结构，压缩树区宽度、节点缩进和上下留白，提升信息密度。
- **动效**: 保留现有 hover/current 态过渡，不新增动画。
- **氛围**: 弱化圆角带来的软卡片感，保留轻量渐变和边界层次，让桌面文件管理观感更硬朗。

### 技术约束
- **可访问性**: 保持现有选中态、hover 态和文字对比度，不缩小到影响辨识
- **响应式**: 仅在现有桌面工作台布局内优化，不引入新的断点逻辑
