## 1. 需求

### 背景
远程文件工作台的文件表格头部在当前紧凑桌面风格下仍显得偏高，尤其在嵌入式 SSH 工作区中会挤占可视内容区域。用户明确要求仅将表头这一行压缩到更紧凑的高度，不调整顶部路径栏或搜索栏。

### 目标
- 将远程文件表格表头从当前高度压缩为更紧凑的视觉尺寸。
- 保持列标题可读性、排序交互和整体桌面文件管理器风格一致。
- 控制改动范围，仅影响远程文件表格头部样式。

### 约束条件
```yaml
时间约束: 本轮直接完成
性能约束: 不引入额外运行时逻辑或响应式状态
兼容性约束: 保持 vxe-table 现有列排序、拖拽和表头布局不变
业务约束: 不调整路径栏、搜索栏、表格行高，仅缩小表头高度
```

### 验收标准
- [ ] 远程文件表格表头高度由 28px 调整为 24px，视觉上明显更紧凑
- [ ] 列标题文字仍垂直居中，排序与列宽拖拽区域不失效
- [ ] 路径栏、搜索栏和表格正文行高保持原状

---

## 2. 方案

### 技术方案
在 `RemoteFileTable.vue` 中继续沿用 CSS 变量控制表头尺寸，将 `--remote-table-header-height` 从 `28px` 下调为 `24px`。保留现有 header flex 对齐与行高逻辑，让表头整体收口而不改动 body 行高和数据渲染逻辑。

### 影响范围
```yaml
涉及模块:
  - ui-components: 远程文件工作台中的文件表格头部样式收紧
预计变更文件: 1
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 表头压缩后标题或拖拽热区变得过窄 | 低 | 仅下调到 24px，保留现有 flex 对齐和列拖拽结构 |
| 视觉密度不一致 | 低 | 保持正文行高不变，只收紧表头，贴合现有紧凑工作台方向 |
